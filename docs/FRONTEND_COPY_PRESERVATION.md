# 前端文案改造保留项

本记录对应《前端整理与修改 (1).md》。该清单基于 `f3dddc7`，当前工作区已经增加导演模式、沉浸剧场、故事上下文压缩和背景资产索引，因此以下内容先保留，避免只删展示文字却破坏后来加入的完整功能。

## 保留项

1. **导演模式与进入前模式选择**
   - 首页新文案已经明确提到“导演剧场”，当前第三步也真实支持自然模式与导演模式二选一。
   - 因此保留第三步的模式选择、导演设定、导演运行面板和导演状态文案；只删除清单明确要求删除的旧自然模式说明。

2. **导演、压缩器与背景资产的必要运行状态**
   - 这些能力晚于清单基线，删除其状态会让用户无法判断导演是否正在编排、公开剧情是否正在整理、背景是否成功复用。
   - 状态改为与其他通知相同的 Toast 外观，底层逻辑、存档字段和运行过程不删除。

3. **来源署名与许可信息**
   - 清单要求删除商用提示，但当前考据链路仍把来源 URL、署名和许可作为已采用资料的一部分。
   - 已删除入口处重复的非商业警告和候选标签中的“非商业使用”，但来源详情中的原页、署名与许可先保留，避免资料进入档案后失去来源对应关系。

4. **关系类型的存档兼容值**
   - 页面显示把“朋友”改为“挚友”、“亲属”改为“挚爱”。
   - 存档内部仍使用原有枚举值，避免旧存档、关系裁判和模式恢复失配；只在选择器和关系边上转换显示名称。

5. **互动会话和资产任务的底层记录**
   - 卷轴现在只展示剧情、行动、对话、后续和角色内心，并隐藏动作校验、空间参数、裁判和资产完成等技术过程。
   - 对应的 Interaction Session、同意裁决、资产索引和存档数据继续保留，舞台中的进行中会话提示也继续承担安全反馈。

6. **后端连接状态**
   - 首页 Footer 已按清单删除，但创建角色依赖后端保存，连接断开时不能静默继续。
   - 因此保留创建向导中的后端连接说明，并把世界内的 API/保存状态统一为 Toast。
